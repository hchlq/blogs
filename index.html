JS/关于逻辑运算符和比较

Posted on 2020-09-25

开发中可能用不到, 但是面试可能常考的有关逻辑运算符的计算

关于如果将常见的各类型转为字符串的规则

  • number: 直接加字符串

  • boolean: 直接加字符串

  • null: 直接加字符串

  • undefined: 直接加字符串

  • 对象: 规则见如下1-1

  • 数组: 规则见如下1-1

关于常见的类型转换为number的规则

  • string: 如果字符串是一个有效的数字，转为数字是有效的如”123” => 123,如果是”123abc”,那么就不是有效的;

  • boolean: true转为1, false转为0;

  • null: 为0

  • undefined: 为NaN

  • 对象: 如{},先将对象转为字符串(规则如1-1)，得到基本类型后强制转为number;为NaN

  • 数组: 先将数组转为字符串(规则如1-1)，由于数组toString()的到的是各元素转成的字符串，如[1, 2, 3] 得到的是”1, 2, 3”。最后进行转换成number类型

逻辑运算符 + - * / % 等

除加号之外的运算符

  • 强制转换为number类型再计算

加号的运算

  • 至少有一个字符串: 这种情况就是进行字符串的拼接
  • 没有字符串，至少有一个对象
    (1-1)先调用该字符串的valueOf()方法, 如果没有返回基本类型，valueOf返回了基本类型，就不会继续往下调用toString()方法，否则继续调用toString()方法。

一种特殊的情况式如果使用Object.create(null);创建的对象，那么该对象没有valueOf和toString方法

自加和自减

  • a++: a自加1, 表达式(a++)返回运算之前的值。
  • ++a: a自加1, 表达式(++a)返回运算之后的值
  • a–: a自减1, 表达式(a–)返回运算之前的值
  • –a: a自减1, 表达式(–a)返回运算之后的值

比较运算符

<= >= < >

  • 两边都是字符串: 从左到右一一进行字符串的ascll码的比较
  • 剩下的转为number类型进行比较，规则见上

== === 相等比较

区别: == 和 === 的区别是前者可以隐式进行类型转换，后者不行

==

  • 相等类型的比较: 原始类型就按原始类型比较，对象则比较引用
  • 不同类型的比较
    • null和undefined: null==undefined， 除此之外，null和undefined不等于任何值
    • 其他类型转为number进行比较，规则见上

细节

  • NaN跟任何类型比较都不相等
  • infinity和-infinity== ===只能和自己比较

  • number string boolean null undefined object symbol

  • typeof中的类型: number string boolean undefind object function symbol

    注意的是，typeof null === 'object' 判断null的类型要用符合条件去判断: (!a && typeof a === 'object')

常量对象

  • 设置常量不可以删改查, 用Object.defineProperty设置writable:false; configurable: false

  • 禁止扩展: 禁止一个对象添加新属性并且保留已有属性

  • 密封: 密封一个对象，会在现有对象上调用Object.preventExtensions(),并把现有属性标记为configurable: false, 密封后不能增和删，但是能改

  • 冻结: 创建一个冻结对象，实际上调用Object,seal(),属性标记为writable: false;,这样久无法修改值，即无法增删改查

小细节

  • null == undefined为true, 但是null === undefined为false;

  • NaN啥也不等，就连NaN === NaN都为false

  • 一个未声明的变量typeof不会报错，得到的是undefined

  • 数组赋值时如果不是用数字索引赋值时，虽然可以访问，但是数组的长度不会变 如:const arr = [] arr['a'] = 'aa'这样得到的数组长度仍然为0,但是可以通过arr[‘a’]访问到值’aa’；特别注意的是，如果arr[‘数字’] = ‘…’ 这样赋值的话，数组的长度就是数字+1了

  • 字符串借用数组的方法处理时有一项是不会改变的

  • 将很大的数字转为指数形式

  • 显示有效的位数

  • js中进制的表示: ob11(二进制) o066(八进制) oxaa(十六进制)

  • 判断一个数字是否值整数

  • js中安全的数字 -(2^53 - 1) –> 2^53 - 1

  • undefined和null的区别:

    • undefined从未赋值过，但是null表示赋值过，但是现在是空值
    • undefined可以当作变量来使用，null不行。但是强烈推荐不要这样做

  • typeof NaN === ‘number'，可称NaN是无效数值，失败数值或者坏数值

  • window.isNaN(num)有一个bug，就是如果是字符串，也返回true, 因此可以使用Es6的Number.isNaN(num)来判断

  • es6提供了工具方法Object.is(n1, n2)一般用来判断两个特殊的值是否绝对相等 判断NaN -0 和 0。能使用 == 和 === （参见第 4 章）时就尽量不要使用 Object.is(..) ，因为前者效率更高、
    更为通用。 Object.is(..) 主要用来处理那些特殊的相等比较

  • 判断一个数是不是NaN的四种有效方法

  • \n对应的十六进制为0x0a，\r对应的十六进制为0x0d。window系统换行是会有\n和\r，但是mac系统只有\r

其他事件

  • load 页面中所有资源全部加载完时触发的事件 使用方式 ：window.onload window.onload一般不会用，一般使用img.onload等音频使用， 因为使用window.onload会等到整个文件全部加载完后才执行 执行时间非常靠后，而且中间如果有什么错误，就会导致不执行

浏览器渲染页面的过程

  • 1 得到页面源代码 2 从上到下执行代码 3 将元素依次添加到dom 树中，每添加一个元素，进行预渲染 资源加载默认情况下是同步的 （与现实生活的同步相反）

JS代码尽量写到body下面 ，要最后加载 ，避免阻塞后续的渲染，因为加载时同步的，防止加载js文件过久，无法加载后续的代码
css尽量写到上面，因为拿到页面时要先看到，避免出现闪烁，因为刚开始得到丑陋的页面，再得到js代码
img video等是异步加载的 当加载img等时可以同时加载其他的下面的代码

  • DomContentloaded : 使用方式要dom2构建 即addEventListener注册 ，document.addEventListener(“DomContentLoaded”,function(){}) dom数构建完成后触发的事件 ，不管其他资源的加载
  • readystate 分为三个阶段 loading interactive(触发DomContentLoaded事件) complete（触发window.onload事件）
  • readystatechange 当阶段改变时触发的事件 即由loading =>interactive =>complete 两次改变 可以用dom0 注册

JS-DOM/详细的获取节点

Posted on 2020-03-22

获取dom节点 获取到多个的都是类数组

获取dom对象

全局对象 window中有属性document,代表整个文档节点

旧的获取元素节点的方式

dom0

  • document.body 获取body元素节点
  • document.documentElement 获取html 元素节点
  • document.head 获取head元素节点
  • document.links 获取页面中所有的超链接节点 即a元素，得到类数组
  • document.anchors 获取页面上所有的锚链接元素节点 a元素中含有name属性节点的元素
  • document.forms 获取所有的form表单元素

dom2 加了s都是类数组

  • document.getElementsByTagName() 通过元素名称获取

  • document.getElementById() 通过id获取元素

  • document.getElmentsByClassName() 通过类名获取 IE9一下无效

  • document.getElementsByName() 通过name属性值获取 一般部分标签生效，表单、img、 frame等

  • document.querySelector() 通过css选择器获取元素 只获得一个 IE8以下无效

  • docuemnt.querySelectorAll() 通过css选择器获取 获得全部

细节:

  • 在所有获的类数组中，除了querySelectorAll() 外，其他否是实时更新的

  • getElementById() 得到元素的执行效率最高

  • 书写了id的元素，会自动成为window对象的属性，它是实时更新的单对象，不推荐使用

  • document.getElementsByTagName , document.getElementsByClassName , querySelector , querySelectorAll可以作为其他节点对象的方法使用 如 var div = docuement.getElementsByTagName() div.document.get…使用

根据节点关系获取节点 会有text 和 comment 节点

  • parentNode 获取元素父节点

  • childNodes 获取子元素们节点 获得类数组

  • firstChild 获取第一个子节点

  • lastChild 获取最后一个子节点

  • previousSibling 获取上一个元素节点

  • nextSibling 获取下一个元素节点

  • attributes 获取属性

获取元素节点

  • parentElement 获取父元素

  • children 获取子元素们 得到类数组

  • firstElementChild 获取第一个子元素

  • lastElementChild 获取最后一个子元素

  • nextElementSibling 获取下一个兄弟元素

  • previousElementSibling 获取上一个兄弟元素

节点类型

  • nodeName 获取节点名称 得到的是大写的

  • nodeValue 获取属性值

  • nodeType 获取节点类型 返回数字 元素节点: 1 属性节点: 2 文本节点: 3
