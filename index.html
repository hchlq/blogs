JS/对象混合和对象克隆

Posted on 2020-03-20

#对象混合：两个对象混合，产生一个新对象 自己封装一个函数，或者使用js提供的方法
-Object.assign(obj1, obj2) obj2混合到到Obj1 把后边的对象混合到前边的对象，返回前边的对象
-改装：Object.assign({}, obj1, obj2) 将obj2和obj1进行混合，混合完后又跟空对象混合,返回那个混合后的空对象
-使用场景，当传入一个对象时，函数内有默认的对象值，将传入的对象与默认值对象进行混合，传入的有值就用传入的值，如果没有值，就是用默认值

#对象克隆 赋值一个完全一样的对象 克隆js没有提供方法 要自己封装
-赋值数组： arr.slice()
-浅度克隆 对对象或数组的属性不做处理，如果里面还有数组，对象等，就会地址一样，修改一个另一个也跟着改变
-深度克隆 对对象和数组的属性再克隆，让他们的地址不一样

JS/错误处理

Posted on 2020-03-20

JS的错误分为两类：

1 语法错误： 会导致整个脚本快无法执行
2 运行错误：

  • 运行报错，会导致后续脚本快无法执行
  • 运行结果不符合预期

调试错误：

-控制台打印： 在程序的各个部分在控制台输出，以此来找到错误
-*断点调试：在谷歌浏览器的sources用断点调试 *

##抛出错误 错误在js中本质上就是对象 抛出错误的语法为
throw newError(“自己写的自定义信息”) ，表示抛出错误对象

错误类型：
-RangeError :数值变量或参数超出有效范围
-ReferenceError: 无效引用
-SyntaxError: 语法错误
-TypeError: 类型错误

##捕获错误
try{
代码块1
}
catch（错误对象）{
代码块2
}
finally{
代码块3
}
当try中发生错误时 ，马上到catch 执行 ，finally无论如何都会执行
错误对象中要可以传 如 传err ==> err.name err.message

JS/this指向

Posted on 2020-03-20

this的指向：函数调用时才知道this的指向

1 在全局作用域中，this指向全局对象window
2 在函数作用域中，this的指向取决于调用的对象
a.直接调用，指向全局对象window 如b();（也叫预编译）
b.通过对象（数组的下标arr[index] 或者是对象{}）的属性调用则指向该数组或对象
c.构造函数创建的对象，函数内部的this指向该新对象
3 箭头函数的this ， 没有自己的this, 找外层是否有函数，如果有，离他最近的外层函数就是它的this, 没有外层函数的话this就指向window

4 call apply 第一个参数就是它的this

一些注意事项：

1 arguements.callee 指向函数本身
2 注意区分console.log(this.name) 和 console.log(name) 前者是找对象，要找到this的指向，后者是访问对象 ，一层一层的找
3 注意在写AO和GO时要考虑this进去，同时找到this的指向
4 new 构造函数时 函数的AO中的隐式this中的变量与函数中的var 变量无关
如 function test() {
var foo = 2;
console.log(this.foo);
}
new test();
中的this.foo 是new新对象的this,与var foo 无关

typeof(Number) 和typeof(new Number)有区别

#Object 用于创建对象，任意对象都是通过Object 创建的
-如果创建时传入的为null 或者underfined 则返回一个空对象，相当于new Object() 什么都不传
-new Object(123) 相当于new Number(123) new Object(“a”)，相当于new String(“a”) new Object(ture)相当于new Boolean(true) new Object([])相当于new Array([])
-new Object({}) 相当于var obj = {}
##静态属性
-Object.prototype 为Object new 出来的对象添加属性和方法，该对象可以继承 因为proto指向Object.prototype
-Object.length 值为1
##静态方法:
-Object.keys(对象) 得到所有属性名组成的数组
-Object.values(对象) 得到所有属性值的数组
-Object.entries(对象)得到属性名和属性值的数组

##实例方法 var obj = {} 或者 var obj = new Object()

***所有的对象，都拥有object的所有实例成员

-obj.toString() 返回该对象的字符串格式 如 利用Object.prototype.toString.call(对象)，并且对象是new 出来的 （如果不是new出来的，直接传Number 就会得到[object Function]）就可以得到 [object Object] [object Array] [object Number] [object String] [object Boolean] Number Boolean Array String 都重写了toString() 方法 var num = new Number(123) ,num.toString()==> “123”
-obj.valueOf() 得到某个对象的值 默认情况下，返回该对象本身
**作用: 当进行自动转换时 ，先调用valueOf() 在调用toString() 如var obj = {a: “aa”} obj + 1 相当于obj.valueOf().toString() 如果调用valueOf() 得到了原始类型（number string boolean undefined），就不会再调用toString() 例如 var obj = {valueOf(){return 111}}
则 obj + 1 ==>112 因为已经的到得到了原始类型 不会在调用toString()

#Function 凡是通过它创建的对象，都是普通函数

**所有的函数都具有Function中所有的实例成员
-关键字arguments 获得所有参数的类数组 类数组没有通过Array创建出来，因此不会拥有Array的实例方法 参数和argumens会一一映射 一个变都会改变

-test.length 得到形参数量
-test.call(xxx，参数1，参数2) *执行test函数 ，test的this指向xxx * 参数一数组传递
-test.apply(xxx,[参数1 ，…]) 执行test函数，test的指向为xxx
