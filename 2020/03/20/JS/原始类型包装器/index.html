JS/原始类型包装器

Posted on 2020-03-20

原始类型包装器

原始类型： number string boolean undefined

Number

  • new Number(值) 返回一个对象 Number{1}
  • Number(值) 返回的是一个原始类型 number

静态成员

-Number.isNaN()或者 window.isNaN (window下的可以直接调用isNaN即可) ， 判断一个数是不是NaN ，请注意isNaN（）和Number.isNaN（）之间的区别：如果当前值是NaN，或者将其强制转换为数字后将是NaN，则前者将返回true。而后者仅当值当前为NaN时才为true：

  • Number.isFinite() 判断是不是有限数,可以在window下使用
  • Number.isInteger() 判断是不是整数 ，包括数后面有多个0也算，不可以在全局下使用
  • Number.parseFloat() 将数据转为小数，全局window下可以使用,可以传第二个参数，第二个参数表示前一个参数数据的进制类型
  • Number.parseInt() 将数据转为整数 ，直接将小数后删掉，去该整数。可以传第二个参数，表示前一个参数的进制类型，将前一个参数按照第二个参数的进制转为整数,如parseInt(3,8),就会将3看成是8进制，将其转换为十进制。全局window下可以使用

**parseInt ,parseFloat 都是从左往右开始看，传入的数据要求是字符串类型，如果不是字符串类型，则会先转换为字符串类型，如果传入空字符串，则返回NaN 如parseInt(“33asd”)则会转为33，后面不管了 但是parseFloat(“aa33”)则会返回NaN

实例成员 var a = 12345 相当于 var a = new Number(12345) a就是实例对象

  • a.toFixed() ，将a精确到小数点后几位
  • a.toPrecision(),将a精确几位，包括整数部分

Boolean （当值为0、null、undefined、NaN 、”” ,则判断为false）

String

**多行字符时 连接用 1.，末尾用 + 2.末尾用\ 3.用模板字符串

静态成员

-String.FromCharCode() , 通过unicode编码创建字符串

实例成员 var str = “akf” 就是 var str = new String(“akf”)

-str.length , 字符串的长度 ，字符串相当于数组，有下标
-str.charAt() ,传入下标，得到相应的下标对应的字母

  • str.charCodeAt() ,得到字符编码 如a => 97
  • str.concat() ，拼接字符串，返回新的字符串
  • str.includes() , str是否包含某个字符串
  • str.endsWith() ,该字符串是否以传入的值结尾
  • str.indexOf() , 返回传入参数对应的索引，从前开始数
  • str.lastIndexOf() ,返回传入参数的对应索引，从后开始数
  • str.padEnd(长度,字符串长度不够时填充的) ，在末尾填充
  • str.padStart(长度，字符串长度不够时填充的) ，在首部填充
  • str.repeat(重复次数)
  • str.slice（下标1，下标2），前开后闭[ ) 下标可以是负数
  • str.substr(起始下标，长度)
  • str.substring(下标1，下标2) 下标不可以是负数 参数位置可以调换
  • str.split() 将字符串分割成为数组
  • str.toLowerCase() 将字符串转换小写
  • str.toUpperCase() 将字符串转换为大写
  • str.trim() 去掉收尾空格
  • str.trimStart()
  • str.trimEnd()
  • str.start()
  • str.start()
