this的指向（由大到小）

  • 默认绑定: 全局调用this，函数直接调用
  • 隐式绑定: 调用函数的上下文对象
  • 显示绑定: 使用call,apply,bind
  • new构造函数: 指向新创建的对象

javaScript中的类型

  • number string boolean null undefined object symbol

  • typeof中的类型: number string boolean undefind object function symbol

    注意的是，typeof null === 'object' 判断null的类型要用符合条件去判断: (!a && typeof a === 'object')

常量对象

  • 设置常量不可以删改查, 用Object.defineProperty设置writable:false; configurable: false

  • 禁止扩展: 禁止一个对象添加新属性并且保留已有属性

  • 密封: 密封一个对象，会在现有对象上调用Object.preventExtensions(),并把现有属性标记为configurable: false, 密封后不能增和删，但是能改

  • 冻结: 创建一个冻结对象，实际上调用Object,seal(),属性标记为writable: false;,这样久无法修改值，即无法增删改查

小细节

  • null == undefined为true, 但是null === undefined为false;

  • NaN啥也不等，就连NaN === NaN都为false

  • 一个未声明的变量typeof不会报错，得到的是undefined

  • 数组赋值时如果不是用数字索引赋值时，虽然可以访问，但是数组的长度不会变 如:const arr = [] arr['a'] = 'aa'这样得到的数组长度仍然为0,但是可以通过arr[‘a’]访问到值’aa’；特别注意的是，如果arr[‘数字’] = ‘…’ 这样赋值的话，数组的长度就是数字+1了

  • 字符串借用数组的方法处理时有一项是不会改变的

  • 将很大的数字转为指数形式

  • 显示有效的位数

  • js中进制的表示: ob11(二进制) o066(八进制) oxaa(十六进制)

  • 判断一个数字是否值整数

  • js中安全的数字 -(2^53 - 1) –> 2^53 - 1

  • undefined和null的区别:

    • undefined从未赋值过，但是null表示赋值过，但是现在是空值
    • undefined可以当作变量来使用，null不行。但是强烈推荐不要这样做

  • typeof NaN === ‘number'，可称NaN是无效数值，失败数值或者坏数值

  • window.isNaN(num)有一个bug，就是如果是字符串，也返回true, 因此可以使用Es6的Number.isNaN(num)来判断

  • es6提供了工具方法Object.is(n1, n2)一般用来判断两个特殊的值是否绝对相等 判断NaN -0 和 0。能使用 == 和 === （参见第 4 章）时就尽量不要使用 Object.is(..) ，因为前者效率更高、
    更为通用。 Object.is(..) 主要用来处理那些特殊的相等比较

  • 判断一个数是不是NaN的四种有效方法

  • \n对应的十六进制为0x0a，\r对应的十六进制为0x0d。window系统换行是会有\n和\r，但是mac系统只有\r

正则表达式

  • (?:xx) 表示匹配不捕获

  • (?=xx) 正向预查,一次正向预查(?=xx)就对整个字符串预

    匹配任何其后紧接指定字符串 xx的字符串。

  • (?!xx) 反向预查 一次正向预查(?!xx)就对整个字符串预查

    匹配任何其后没有紧接指定字符串 n 的字符串。
