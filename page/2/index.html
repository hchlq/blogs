JS-DOM/dom元素的操作

Posted on 2020-03-22

dom元素操作

获取和设置元素的属性

  • 通用方式：getAttribute() setAttribute(属性名，改的值) ===>一般用于自定义属性

细节： 原始写到元素上的，不可以不赋值
一般用用于自定义的属性

可识别属性

  • 正常的html属性 如input 的value，type等

  • dom对象.属性名 推荐使用该方法操作html属性

细节： 正常的属型即使没有赋值（即没有直接在元素上写），但是也有默认值
某些表单元素可以获取到该元素不存在的属性
某些属性的名字与js的保留字冲突 （如 class==>className、for==>htmlFor）
布尔属性在dom对象中，得到的是boolean

自定义属性

  • 自定义属性使用** data-属性名 = “ xxx” 为格式 即以data作为前缀，遵从该规范后可以用 dom对象.dataset.属性名获取
    如
    获取dom对象后就可以使用document.getElementsByTagName

删除属性名

  • removeAttribute(“属性名”)
  • delete dom.dataset.属性名 建议使用这种

设置和获取元素的内容

  • dom对象.innerHTML，获取和设置dom对象的文本内容 可以添加元素
  • dom对象.innerText，获取纯文本，仅得到内部显示出来的文本，添加元素时会成为文本，不会成为元素
  • dom对象.textContent,得到内部所有源代码的文本，包括隐藏的元素也可以得到该文本(IE浏览器)
  • 可以将style写到body中，将style的样式设为display：block white-space: pre 和加上属性 contenteditable= “true” ,就可以将该style显示在页面上 ,并且可以修改

元素结构重构

  • 父元素.appendChild(加入的元素) : 在该元素的末尾加上一个子元素(只能一个)
  • 父元素.append(加入的元素) 可以加多个 不建议使用
  • 父元素.insertBefore(插入的元素,插入的元素之前)
  • 父元素.replaceChild(替换的元素 ，被替换的元素)

创建和删除元

创建 document.createxxx()

创建元素

  • document.createElement(“元素名称”)

  • document.createDocumentFlagment() 相当于一个容器，无参数

克隆元素

  • dom对象.cloneNode(是否深度克隆) 赋值一个新的dom对象 如果是深度克隆，则将元素的文本内容等都赋值过来

删除元素

-父元素.removeChild(父元素的子元素) 删除后对象还在，只是在页面消失而已（”谋杀”）
-dom对象.remove() ,无参数，删除自己（”自杀”）

CSS/有价值的css细节

Posted on 2020-03-22

  • 一个盒子的包含块，就是父元素的内容区，即(width + height) , 该盒子的宽（width + padding + border + margin）相加一定等于父元素的内容区，即总宽度占满父元素的内容区 width 和margin为auto时 width 吸收能力大于 margin

  • 常规流: 块盒独占一行，行盒一次排列

  • 行盒的宽高取决于内容的宽高

  • 盒子的组成出height除外，百分比设置都是相对于父元素的宽度

  • 当一个盒子浮动后，一定为block

  • 浮动元素的包含块，与常规流一样，为父元素的内容盒。

  • 浮动元素的宽度和高度为auto时，适应内容的高度，margin为auto时为0

  • reletive没有脱离常规流，absolute和fixed脱离了常规流

  • 定位元素的包含块是父元素的填充盒 padding + content

  • 常见的块盒: 容器元素 、h1-h6、 p元素 form adress

  • 常见的行盒: span strong em del a i

CSS/有关文字环绕图片的细节

Posted on 2020-03-22

有关文字环绕图片的细节：

  • 图片是左浮动或则右浮动，文字就会围绕在其周围，包裹文字的容器如果是块级元素会避开图片，因为文字默认是用一个行盒包裹，行盒会避开浮动盒子，但是包裹容器的还是块级元素，因此它的实际距离仍然是到它父元素的边距，如果想设置图片和文字的距离，要设置图片的margin-left(图片是左浮动的情况下)或设置图片的margin-right(图片是右浮动的情况下) 因为如果设置块级的margin-left/right,他是从它父级的边距距离开始算

CSS/水平垂直居中总结

Posted on 2020-03-22

css的水平垂直居中总结(都是有包含块的前提之下)

浮动元素没有水平垂直居中

水平居中：

  • 单行文本的水平居中：在其父元素加 text-align: center;
  • 常规流块盒的水平居中： 本身使用 已知宽 +margin: 0 auto;(常规流块盒不可以垂直居中)
  • 定位元素的水平居中（absolute 和 fixed） left:0 right:0 margin: 0 auto

垂直居中：

  • 单行文本： 设置 line-height: 等于父级的 height
  • 多行文本： 在文本再包上容器，让该容器变为定位 或者文本的父级使用 padding 设置成垂直居中的样式
  • 定位元素的垂直居中：
    一 定高 top : 0 bottom: 0 margin: auto 0;
    二 定高 top: 50% margin-top: -height;
    三 定高 top: 50% transform: translate(0, -50%); 高度不确定也行，高度可以任意改变

目前关于定位元素的水平垂直居中的三个方案

要定死宽高的 height: 200px; width: 200px; position：absolute; top: 50%; left:50%; margin-left: - 100px(二分之一宽); margin-top: -100px（二分之一高）;

宽高自适应的，不用定死 可以自定义 ：
1 position: absolute; left: 0; bottom: 0; left: 0; right: 0; margin: auto;

2 position: absolute; left: 50%; top: 50%; transform: translate(-50%, -50%)
