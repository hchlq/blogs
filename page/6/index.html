JS/对象混合和对象克隆

Posted on 2020-03-20

#对象混合：两个对象混合，产生一个新对象 自己封装一个函数，或者使用js提供的方法
-Object.assign(obj1, obj2) obj2混合到到Obj1 把后边的对象混合到前边的对象，返回前边的对象
-改装：Object.assign({}, obj1, obj2) 将obj2和obj1进行混合，混合完后又跟空对象混合,返回那个混合后的空对象
-使用场景，当传入一个对象时，函数内有默认的对象值，将传入的对象与默认值对象进行混合，传入的有值就用传入的值，如果没有值，就是用默认值

#对象克隆 赋值一个完全一样的对象 克隆js没有提供方法 要自己封装
-赋值数组： arr.slice()
-浅度克隆 对对象或数组的属性不做处理，如果里面还有数组，对象等，就会地址一样，修改一个另一个也跟着改变
-深度克隆 对对象和数组的属性再克隆，让他们的地址不一样

JS/错误处理

Posted on 2020-03-20

JS的错误分为两类：

1 语法错误： 会导致整个脚本快无法执行
2 运行错误：

  • 运行报错，会导致后续脚本快无法执行
  • 运行结果不符合预期

调试错误：

-控制台打印： 在程序的各个部分在控制台输出，以此来找到错误
-*断点调试：在谷歌浏览器的sources用断点调试 *

##抛出错误 错误在js中本质上就是对象 抛出错误的语法为
throw newError(“自己写的自定义信息”) ，表示抛出错误对象

错误类型：
-RangeError :数值变量或参数超出有效范围
-ReferenceError: 无效引用
-SyntaxError: 语法错误
-TypeError: 类型错误

##捕获错误
try{
代码块1
}
catch（错误对象）{
代码块2
}
finally{
代码块3
}
当try中发生错误时 ，马上到catch 执行 ，finally无论如何都会执行
错误对象中要可以传 如 传err ==> err.name err.message
