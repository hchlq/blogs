JS/正则表达式

Posted on 2020-03-20

基础

正则表达式是默认贪婪匹配的 如. 去匹配123 会分别得到1 2 3

  • . 任意字符 ，一个点只能表示一个

  • ^ 开头标志，在[^]中表示非的意思

  • $ 结尾字符

  • \n 转行字符

  • \t 制表符

  • \s 空白字符 （制表符，空格，换行等）

  • \S 非空白字符 [\s\S]可以匹配所有

  • \b 单词边界 一个词有空格等隔开 如\beee 表示eee前有空格或eee是开头隔开

  • \B 非单词边界

  • \d 匹配字符 0-9 贪婪匹配

  • \D 非字符

  • \w 匹配字母 数字 下划线 等价于[A-Za-z0-9]

  • \W 匹配非数字 字母 下滑线

  • \uxxxx 每个字都有四个十六进制 unicode编码

字符集

  • []里边的表示范围
  • [a-z]
  • [A-Z]
  • [0-9]
  • [a-zA-Z0-9] 相当于\w
  • [^字符范围] 非该字符范围
  • [\d\D]+ , [\s\S]+匹配所有字符

中文字符集 4e00-9fa5
-[\u4e00-\u9fa5]

量词***

  • * 匹配0个到多个

  • + 匹配1个到多个

  • ？匹配0个或者1个

-{n} 匹配n个

-{n,} >=n个 即至少n个

-{n,m} 匹配n-m个

或者

多个规则之间，用 |

正则表达式进阶

***特别注意的是：在正则表达式中一般都写g 表示全局匹配

捕获组 也叫子表达式 （想要获得捕获组，要使用reg.exec(str)）

  • 正则表达式中用小括号括起来的部分就叫捕获组，捕获组会出现在匹配结果中（使用reg.exec(str)才有）

  • 给捕获组命名：在括号中使用’ ?<名字>’ 命名后就有group（调用reg.exec(str)的情况下）

  • 非捕获组: ‘ ?: ‘ （?: abc）只是当成一个小括号，一个整体而已，不用捕获得到这个捕获组的结果

  • 使用： a 分类 b str.replace(reg, function($,$1, $2…){}) $是整个正则表达式得到的字符串 ，$1等是正则表达式捕获组 也可以str.replace( reg , ‘$1/$2/$3’) $1表示第一个捕获组 …

反向引用，在正则表达式中，使用某个捕获组，使用方式 ‘(捕获组)\捕获组编号’

正向预查 检查某个字符后面是否满足某个规则，该规则不成为匹配结果 ,该捕获组不会成为结果

  • （?=xxx）

负向预查

检查某个字符后面是否不满足某个规则，该规则不成为匹配结果，该捕获组不会成为结果

  • （?!xx）

正则表达式在js的运用

在js中,正则表达式表现为一个对象，该对象是通过构造函数创建的

创建对象 var reg = /正则/匹配配置 知道规则 即规则固定

var reg = new RegExp('正则',[匹配配置])      不清楚规则的情况下
var reg = RegExp('正则',[匹配配置])

匹配配置

  • g 全局匹配 global

  • i 忽略大小写 ignore

  • m 多行匹配

##正则实例成员 var reg = /正则/ 等同于 var reg = new RegExp()

  • reg.test(str) 验证字符串是否满足规则 开启全局匹配时 会有连续性 接着上一次匹配的索引 利用此特性，可以判断匹配了多少次 var n = 0 while(reg.test(str)){n ++}

  • reg.lastIndex 配合全局匹配下的索引

只有量词时，正则表达式默认情况下是贪婪模式（尽可能多） 取消贪婪模式时 在量词(+?等)后使用?

  • reg.exec(str) 执行匹配，得到匹配结果 得到真数组 index表示得到的结果在该字符串的下标

  • reg.global 是否开启全局匹配

  • reg.ignoreCase 是否开启忽略大小写

  • reg.multiline 是否开启多行匹配

  • reg.source 得到当前匹配规则

##字符串对象中的正则 var str = ‘aaa’

  • str.match( reg ) 得到匹配结果

  • str.replace ( reg, 替换得到正则结果的内容（主要是函数其作用 function ( $, $1, $2,$… ) { } ） ) $代表正则表达式的字符串，$1 代表第一个捕获组 $2 代表第二个捕获组 ….

  • str.search ( reg ) 搜索得到满足条件的第一个结果的下标

  • str.split ( reg， [limit] ) 按照给定的正则对字符串进行分割 得到匹配结果的数组,limit可以指定分割的数量

JS/原型

Posted on 2020-03-20

原型

1 默认情况下，prototype 是object
2 默认情况下，constructor指向构造函数本身
3 普通对象没有prototype 既 通过函数或Object ，Array new的对象，
4 所有的对象都有隐式原型proto 介于中间层的如 Array 自定义函数 等，它们的隐式原型指向Object原型

原型方法

Object上的方法，只能由Object调用 普通对象不能用，因为继承不了
1 Object.create(普通对象或者对象的prototype) ，以该对象的原型作为创建的新对象的隐式原型 proto指向
2 Object.getPrototypeOf(对象) ，获取传入对象的隐式原型

3 A instanceof B , 判断B的原型是否在A的原型链上

Object.prototype上的方法，可以是普通对象，因为会继承Object.prototype上的方法
4 Object.prototype.isprototypeOf(对象)，判断当前对象是否在指定对象（Object.prototype）的原型链上
5 Object.prototype.hasOwnProperty(属性名) ，判断该属性是否在指定对象（Object.prototype）上
指定对象不一定是Object.prototype,可以是其子对象

原型的应用：

1 将类数组转为真数组 （[].slice.call(类数组) Array.prototype.slice.call(类数组) Array.from(类数组)）
2 实现继承 如 function User(){} function Vuser(){} 原来Vuser和User的prototype都指向Object.prototype,现在想要Vuser的prototype指向User的prototype

原始类型包装器

原始类型： number string boolean undefined

Number

  • new Number(值) 返回一个对象 Number{1}
  • Number(值) 返回的是一个原始类型 number

静态成员

-Number.isNaN()或者 window.isNaN (window下的可以直接调用isNaN即可) ， 判断一个数是不是NaN ，请注意isNaN（）和Number.isNaN（）之间的区别：如果当前值是NaN，或者将其强制转换为数字后将是NaN，则前者将返回true。而后者仅当值当前为NaN时才为true：

  • Number.isFinite() 判断是不是有限数,可以在window下使用
  • Number.isInteger() 判断是不是整数 ，包括数后面有多个0也算，不可以在全局下使用
  • Number.parseFloat() 将数据转为小数，全局window下可以使用,可以传第二个参数，第二个参数表示前一个参数数据的进制类型
  • Number.parseInt() 将数据转为整数 ，直接将小数后删掉，去该整数。可以传第二个参数，表示前一个参数的进制类型，将前一个参数按照第二个参数的进制转为整数,如parseInt(3,8),就会将3看成是8进制，将其转换为十进制。全局window下可以使用

**parseInt ,parseFloat 都是从左往右开始看，传入的数据要求是字符串类型，如果不是字符串类型，则会先转换为字符串类型，如果传入空字符串，则返回NaN 如parseInt(“33asd”)则会转为33，后面不管了 但是parseFloat(“aa33”)则会返回NaN

实例成员 var a = 12345 相当于 var a = new Number(12345) a就是实例对象

  • a.toFixed() ，将a精确到小数点后几位
  • a.toPrecision(),将a精确几位，包括整数部分
